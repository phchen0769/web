<pre> 与 <code> 标签的区别

理解这两个HTML标签的不同用途和最佳实践

概述

<pre> 和 <code> 都是HTML中用于展示代码的标签，但它们有不同的用途和特性。了解它们的区别有助于在正确的情境下使用它们。

<pre> 标签

定义：预格式化文本元素

作用：保留文本中的空格、换行和制表符，按原样显示内容

示例：

function hello() {
    console.log("Hello, World!");
    return true;
}

上面的代码保持了原有的缩进和换行格式。

特点：

  • 保留所有空白字符
  • 通常使用等宽字体显示
  • 适合显示代码块、诗歌、ASCII艺术等
  • 默认情况下会有外边距
<code> 标签

定义：内联代码元素

作用：表示计算机代码片段，通常用于段落中的代码引用

示例：

在JavaScript中，可以使用 console.log() 方法向控制台输出信息。

HTML标签如 <div> 和 <span> 是常用的容器元素。

特点：

  • 不保留空白字符格式
  • 通常使用等宽字体显示
  • 适合在段落中标记代码片段
  • 默认情况下是内联元素

结合使用 <pre> 和 <code>

在实际开发中，我们经常将这两个标签结合使用，以展示格式化的代码块：

HTML代码：

<pre>
<code>
function calculateSum(a, b) {
    // 这是一个求和函数
    return a + b;
}
</code>
</pre>

显示效果：

function calculateSum(a, b) {
    // 这是一个求和函数
    return a + b;
}

这种组合方式既保持了代码的格式（通过<pre>），又语义化地标记了代码内容（通过<code>）。
